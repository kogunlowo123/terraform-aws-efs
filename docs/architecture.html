terraform-aws-efs

Architecture Document — AWS Elastic File System Module

Executive Summary

Terraform module to provision an AWS Elastic File System (EFS) with mount targets across multiple availability zones, configurable access points with POSIX user and root directory settings, cross-region replication, automatic backup policies, and lifecycle policies for transitioning data to Infrequent Access storage. The module includes a dedicated NFS security group with VPC CIDR-scoped ingress on port 2049, encryption at rest enabled by default with optional customer-managed KMS keys, and supports both generalPurpose and maxIO performance modes.

Data Flow

1
EC2 instances, ECS tasks, or Lambda functions connect to EFS via mount targets in their respective availability zones.
2
NFS traffic on port 2049 is allowed through the security group scoped to the VPC CIDR block.
3
Access points enforce POSIX user identity and root directory isolation for multi-application file sharing.
4
Lifecycle policy transitions infrequently accessed files to the IA storage class for cost optimization.
5
Data at rest is encrypted using AWS-managed or customer-managed KMS keys.
6
AWS Backup creates automatic backups of the file system for point-in-time recovery.
7
Cross-region replication continuously replicates file system data to a destination region for disaster recovery.

Security Controls

Encryption at Rest

Enabled by default (encrypted = true) with optional customer-managed KMS key via kms_key_arn.

NFS Security Group

Dedicated security group restricts NFS traffic (port 2049) to the VPC CIDR block only. Additional security groups can be layered.

Access Point Isolation

Access points enforce POSIX user/group identity and root directory boundaries, providing application-level file system isolation.

Automatic Backups

AWS Backup integration enabled by default (enable_backup = true) for continuous data protection.

In-Transit Encryption

EFS supports TLS encryption in transit when mounted with the -o tls option via the EFS mount helper.

Industry Adaptation

Healthcare

KMS encryption for PHI at rest. Access points for HIPAA application isolation. Cross-region replication for disaster recovery compliance.

Finance

Automatic backups for financial data retention requirements. Provisioned throughput for predictable performance on report generation workloads.

Government

CMK encryption for data sovereignty. Security group scoping for network segmentation. Lifecycle policies for records management cost control.

Retail

Shared file storage for content management systems. Elastic throughput for seasonal traffic. IA lifecycle for infrequently accessed media assets.

SaaS

Access points for multi-tenant file isolation. generalPurpose mode for latency-sensitive workloads. Cross-AZ mount targets for high availability.

Production Readiness Checklist

  • Enable encryption at rest with a customer-managed KMS key for sensitive data
  • Deploy mount targets in at least two availability zones for high availability
  • Configure lifecycle policies to transition cold data to Infrequent Access storage
  • Enable automatic backups for data protection and compliance
  • Use access points to enforce POSIX user identity for multi-application sharing
  • Restrict security group ingress to specific CIDR blocks or security groups
  • Configure cross-region replication for disaster recovery requirements
  • Choose appropriate performance mode (generalPurpose vs maxIO) based on workload
  • Set provisioned throughput for workloads requiring predictable I/O performance
  • Mount with TLS encryption in transit using the EFS mount helper
